# Твой психолог — Веб-приложение для психологической поддержки

## 1. О проекте

### 1.1. Назначение

Веб-приложение «Твой психолог» разработано в рамках курсовой работы по веб-разработке. Продукт представляет собой комплексную платформу для оказания психологической поддержки пользователям, сочетающую инструменты самостоятельной работы (упражнения, медитации, дневник настроения) с возможностью получения профессиональной помощи.

### 1.2. Актуальность

Современное общество характеризуется высоким уровнем стресса, тревожности и эмоционального выгорания. Согласно данным Всемирной организации здравоохранения, распространенность психических расстройств за последние десять лет увеличилась на 25%. При этом доступность квалифицированной психологической помощи остается ограниченной. Цифровые решения в области ментального здоровья позволяют:

- Обеспечить круглосуточный доступ к базовой поддержке
- Снизить порог входа для обращения за помощью
- Разгрузить специалистов за счет автоматизированных инструментов
- Предоставить пользователю инструменты для самопомощи

### 1.3. Цель разработки

Создание функционального веб-приложения, демонстрирующего комплексный подход к поддержке психологического здоровья, включающий:

- Техники саморегуляции и управления стрессом
- Инструменты самопознания и рефлексии
- Элементы поддержки на основе искусственного интеллекта
- Механизмы взаимодействия с профессиональными психологами

### 1.4. Статус разработки

Представленная версия является демонстрационной. Она предназначена для ознакомления с функциональными возможностями, интерфейсом и архитектурой будущего полноценного продукта.

Инструмент для систематического отслеживания эмоционального состояния пользователя. Функциональные возможности:

Ознакомительный раздел, содержащий описания тестовых методик:

В демонстрационной версии тесты представлены в ознакомительном режиме без возможности прохождения.

### 2.5. Чат с нейросетью

Интерактивный чат с ассистентом, имитирующим работу нейросети. Система анализирует сообщения пользователя по ключевым словам и генерирует релевантные ответы. Поддерживаемые тематики:

- Приветствия и общие вопросы
- Эмоциональные состояния (тревога, стресс, паника, волнение)
- Практические техники (дыхательные упражнения, медитации)
- Вопросы обращения к психологам
- Тематика сна и отдыха
- Выражение благодарности

Раздел с информацией о специалистах, предоставляющих консультации. Каждая карточка содержит:

## 3. Техническая архитектура

### 3.1. Технологический стек

| Компонент | Технология | Обоснование выбора |
|-----------|------------|-------------------|
| Серверная часть | PHP 8 | Широкая распространенность, простота развертывания, полная поддержка объектно-ориентированного программирования |
| Клиентская часть | HTML5, CSS3, JavaScript | Кроссплатформенность, отсутствие необходимости в компиляции |
| Формат взаимодействия | REST API | Универсальность, простота отладки, независимость клиентской и серверной частей |
| Хранение данных | Сессии PHP | Упрощенное хранение данных в демонстрационной версии |

### 3.2. Архитектура приложения

Приложение реализовано по модели «единого файла» (single-file application), что обеспечивает:

- Упрощенное развертывание без необходимости в настройке дополнительных компонентов
- Легкость переноса между различными хостинговыми средами
- Удобство демонстрации функциональности

Взаимодействие между клиентской и серверной частями организовано по следующей схеме:

1. Клиент отправляет HTTP-запрос к серверу
2. Сервер определяет тип запроса (API-запрос или запрос страницы)
3. При API-запросе выполняется соответствующая бизнес-логика, формируется JSON-ответ
4. При запросе страницы сервер возвращает HTML-документ с интерфейсом

| GET | /api/audio | Получение списка аудиоматериалов | Да |
| GET | /api/tests | Получение списка тестов | Да |
| GET | /api/psychologists | Получение списка психологов | Да |

### 3.4. Система аутентификации

Аутентификация пользователей реализована на основе токенов:

1. При успешной регистрации или входе сервер генерирует уникальный токен
2. Токен сохраняется в localStorage клиента
3. При каждом API-запросе токен передается в заголовке Authorization
4. Сервер проверяет валидность токена перед выполнением защищенных операций

В демонстрационной версии используется фиксированный токен для упрощения тестирования.
